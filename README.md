# 对照 (MatchDiff)

`MatchDiff` 是一款轻量级的纯前端文件对比工具。它旨在通过浏览器本地解析技术，帮助用户直观地核对两个文件之间的内容差异，支持表格、文档及纯文本格式。

## 🛠️ 主要功能

  * **多格式解析**：支持 `.xlsx`, `.xls`, `.docx`, `.pdf` 及 `.txt` 格式文件的内容读取。
  * **双模式对比**：
      * **表格模式 (Excel)**：基于单元格坐标进行逐格比对，差异单元格以红色背景高亮显示，并支持表头滚动冻结。
      * **文本模式 (PDF/Word/Txt)**：提取全文内容进行逐行比对，通过背景色和删除线标注内容的增减。
  * **同步视野**：左右双窗格联动滚动，确保对比位置始终保持一致。
  * **本地处理**：所有文件解析与对比逻辑均在浏览器本地完成，数据不上传服务器，保障信息私密性。

## 📖 使用方法

1.  在左侧上传框内拖入或选择“原件”。
2.  在右侧上传框内拖入或选择“修订件”。
3.  点击中间的 **「执行核对」** 按钮。
4.  在下方预览区查看结果。在表格模式下，**双击列号 (A, B, C...)** 可根据内容自适应调整列宽。

## ⚠️ 已知局限与缺陷

在决定使用本工具前，请知晓以下目前技术手段尚无法完全解决的缺陷：

1.  **扫描件识别限制**：
      * 本工具**不支持 OCR（光学字符识别）**。对于扫描生成的图片 PDF 或照片，无法提取其中的文字，仅能识别原生电子版 PDF。
2.  **复杂排版丢失**：
      * 为了实现对比，工具会提取文件中的核心数据或文本。因此，原文件中的图片、图表、复杂的嵌套表格样式、特殊字体格式等无法在预览区原样还原。
3.  **大文件性能表现**：
      * 由于对比计算消耗客户端内存，当处理超大规模数据（如数万行的 Excel 或数百页的文档）时，可能会出现页面响应卡顿。
4.  **坐标对齐逻辑**：
      * 表格对比基于行号与列号的物理坐标。如果其中一份文件在中间插入了新行，会导致后续所有坐标错位而出现大面积差异报警。

  * [Mammoth.js](https://www.google.com/search?q=https://github.com/mwilliamson/mammoth.js) - Word 解析
  * [jsdiff](https://github.com/kpdecker/jsdiff) - 文本差异算法
